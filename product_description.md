# Chrome浏览器多页面管理工具产品说明

## 1. 产品概述

本产品是一款专为Chrome浏览器用户设计的多页面管理工具，旨在解决用户在日常使用中面临的标签页过多、重复页面泛滥、窗口管理混乱等问题，帮助用户提高浏览器使用效率，减少在不同页面和窗口间切换的时间浪费。

## 2. 用户痛点分析

### 2.1 标签页过多问题
- **行为习惯**：用户有不停新开标签页的习惯
- **问题表现**：标签页数量过多，难以快速找到所需页面
- **影响**：降低工作效率，增加认知负担

### 2.2 重复页面泛滥
- **行为习惯**：用户经常忘记已打开过某个页面，重复打开
- **问题表现**：同一网站或页面在多个标签页中重复出现
- **影响**：占用系统资源，增加标签页管理难度

### 2.3 窗口管理混乱
- **行为习惯**：用户喜欢开很多浏览器窗口
- **问题表现**：多个窗口同时存在，需要在不同窗口间频繁切换
- **影响**：浪费时间，降低工作效率

### 2.4 仪表盘唤起路径繁琐
- **行为习惯**：用户需要快速查看所有标签页状态
- **问题表现**：需要点击插件图标 → 再点击"仪表盘"按钮，路径过长
- **影响**：打断工作流，降低使用意愿

### 2.5 信息密度不足
- **行为习惯**：用户需要在一个视图中快速浏览多个标签页
- **问题表现**：现有布局卡片过大，一屏可展示内容有限
- **影响**：需要频繁滚动，降低查找效率

## 3. 已实现功能

### 3.1 统一仪表盘功能
- **功能描述**：提供一个集中式的仪表盘界面，展示用户所有打开的页面
- **核心能力**：
  - 实时显示所有浏览器窗口和标签页的状态
  - 支持快速定位和切换到任意页面
  - 支持拖拽标签页在窗口间移动
  - 支持关闭单个标签页、关闭整个窗口、关闭其他窗口

### 3.2 仪表盘多布局模式（新增）
- **功能描述**：提供三种不同的布局模式，满足不同场景的浏览需求
- **布局选项**：
  - **卡片网格**（默认）：以卡片形式展示标签页，包含图标、标题、URL、操作按钮
  - **紧凑列表**：以单行列表形式展示，隐藏URL，信息密度最高，适合快速扫描
  - **杂志翻页**：一次只展示一个窗口的内容，支持左右方向键翻页，适合专注查看单个窗口
- **用户价值**：
  - 根据场景选择合适的浏览方式
  - 提高信息密度，减少滚动操作
  - 支持布局记忆，下次打开自动恢复

### 3.3 实时搜索功能（新增）
- **功能描述**：支持按标签页标题和URL实时过滤
- **核心能力**：
  - 仪表盘顶部搜索框：输入即过滤，支持 `⌘K`（Mac）/ `Ctrl+K`（Windows）快捷键聚焦
  - Popup 内搜索框：点击插件图标后自动聚焦，实时过滤标签页
- **用户价值**：
  - 标签页数量多时快速定位目标页面
  - 减少手动滚动查找时间

### 3.4 快捷唤起仪表盘（新增）
- **功能描述**：提供多种方式快速打开仪表盘
- **唤起方式**：
  - **全局快捷键**：Mac `Control+D` / Windows `Alt+Shift+D`（可在扩展管理页面自定义）
  - **Popup 一键直达**：点击插件图标后，"打开仪表盘"按钮高亮显示，底部显示快捷键提示
  - **Popup 内搜索**：点击插件图标后自动聚焦搜索框，可直接搜索标签页
- **用户价值**：
  - 减少操作路径，快速进入仪表盘
  - 不打断当前工作流

### 3.5 重复页面检测与管理
- **功能描述**：自动检测并管理重复打开的页面
- **核心能力**：
  - 实时检测已打开的页面URL
  - 识别重复页面并以红色边框标识
  - 提供"清理重复"一键关闭功能
  - 支持设置重复页面处理策略（仅提示 / 自动关闭）
- **用户价值**：
  - 减少系统资源占用
  - 降低标签页管理复杂度
  - 避免在重复页面间切换

### 3.6 快速切换与操作
- **功能描述**：提供多种快捷操作方式
- **核心能力**：
  - **双击 Command 快速切换**：在仪表盘页面内，快速双击 Command 键可随机切换到其他标签页
  - **点击即切换**：点击任意标签页卡片即可切换到该页面
  - **批量操作**：关闭重复页面、关闭其他窗口等
- **用户价值**：
  - 减少鼠标操作，提高效率
  - 适合需要频繁切换标签页的场景

### 3.7 高频页面快速访问（新增）
- **功能描述**：自动统计用户高频打开的页面，提供一键快速访问入口
- **核心能力**：
  - **访问频次统计**：监听标签页创建事件，按URL维度累计打开次数，持久化存储到 `chrome.storage.local`
  - **高频页面展示**：在仪表盘顶部和Popup中展示Top N高频页面列表，显示页面图标、标题、打开次数
  - **一键直达**：点击高频页面条目即可在新标签页中打开对应URL
  - **智能衰减**：访问频次按时间衰减（近期访问权重更高），避免历史高频但已不再使用的页面长期占据列表
  - **手动管理**：支持从高频列表中移除不需要的页面，支持固定/置顶常用页面
- **用户价值**：
  - 无需搜索即可快速访问常用页面，减少重复输入URL或翻找书签的时间
  - 基于真实使用习惯的个性化推荐，越用越精准
  - 与重复页面检测联动：高频页面更易被重复打开，结合重复检测可引导用户直接切换到已打开的标签页

## 4. 技术实现思路

### 4.1 Chrome扩展架构
- **实现方式**：基于Chrome Extension API开发浏览器扩展
- **核心API**：
  - `chrome.tabs`：标签页管理（创建、关闭、移动、更新）
  - `chrome.windows`：窗口管理（获取、创建、关闭、更新焦点）
  - `chrome.storage`：数据存储（用户设置、布局偏好）
  - `chrome.runtime`：扩展通信（Popup、Background、Dashboard 间通信）
  - `chrome.commands`：全局快捷键

### 4.2 页面检测与分组
- **URL解析**：提取域名、路径、参数等信息
- **域名分组**：Popup 中按域名对标签页进行分组展示
- **实时刷新**：仪表盘每3秒自动刷新，保持数据同步

### 4.3 重复页面检测
- **URL匹配**：精确匹配URL识别重复页面
- **实时监控**：监听标签页创建和更新事件
- **视觉标识**：重复页面以红色边框标识

### 4.4 高频页面统计与推荐
- **数据采集**：在 `background.js` 中监听 `chrome.tabs.onCreated` 事件，提取URL并累计计数
- **数据结构**：存储格式为 `{ url, title, favIconUrl, count, lastVisitedAt, pinned }` 的数组，按 `count` 降序排列
- **时间衰减**：每次统计时对历史记录应用衰减因子（如 `count = count * 0.95`），使近期行为权重更高
- **数据清洗**：过滤 `chrome://`、`chrome-extension://` 等内部页面；合并同一域名下仅参数不同的URL（可配置）
- **存储策略**：使用 `chrome.storage.local` 持久化，最多保留 Top 100 条记录，减少存储占用
- **展示联动**：高频页面列表在仪表盘和Popup中共享同一数据源，通过 `chrome.runtime.sendMessage` 实时同步

## 5. 用户体验设计

### 5.1 界面设计
- **简洁直观**：仪表盘界面清晰展示所有页面
- **操作便捷**：支持拖拽、点击切换等常用操作
- **视觉反馈**：重复页面（红边）、当前页面（绿边）有明确视觉标识
- **三种布局**：卡片网格、紧凑列表、杂志翻页，满足不同浏览需求
- **高频页面专区**：仪表盘顶部和Popup顶部展示高频页面横条列表，图标+标题+频次标签，紧凑排列

### 5.2 交互设计
- **智能提示**：在用户可能打开重复页面时提前提示
- **批量操作**：支持关闭重复页面、关闭其他窗口等
- **个性化设置**：允许用户自定义重复页面处理策略、快捷键偏好等
- **布局记忆**：自动保存用户选择的布局模式
- **高频页面交互**：点击打开页面；右键或长按显示操作菜单（置顶、移除、复制URL）；若目标页面已打开则引导切换而非重复打开

### 5.3 快捷操作
- **全局快捷键**：快速打开仪表盘、关闭重复页面
- **⌘K 搜索**：在仪表盘内快速聚焦搜索框
- **双击 Command**：在仪表盘内快速切换标签页
- **方向键翻页**：杂志模式下支持左右方向键翻页

## 6. 预期效果

### 6.1 效率提升
- 减少标签页数量：通过重复页面检测和合并，减少标签页总数

- 加速常用页面访问：通过高频页面列表，一键直达常用网站，无需搜索或翻找书签

### 6.2 用户体验改善
- 减少认知负担：清晰的页面结构和分组管理
- 提高专注度：减少在不同页面和窗口间的频繁切换
- 个性化体验：根据用户习惯优化管理策略和布局偏好
